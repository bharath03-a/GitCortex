Does a code knowledge graph help AI work more efficiently?

Measured Claude token usage on real developer questions, once with ordinary file search, once with the GitCortex graph (compact MCP). This is the median of 3 runs with errored/rate-limited sessions excluded — the noise-resistant view. Every number is from the actual API response.

What shipped in v0.6.3

  • Compact-by-default: MCP server now emits the compact response format unless --full is passed — eliminates ~14k tokens/turn of schema overhead that was hurting TypeScript/Go/Java scores.
  • Confidence-tagged callers & subgraph: find_callers and get_subgraph now annotate each result with (X direct, Y inferred) so the model can weight cross-file edges appropriately.
  • find_cycles: new MCP action exposes Tarjan SCC — returns strongly-connected call cycles ranked by size, capped at 10k import edges.
  • Community-grouped start_tour: tour clusters symbols by call-graph community before ranking, giving a map-of-the-codebase view instead of a flat popularity list.
  • Staleness guard: every MCP response now checks last_indexed_sha vs HEAD + dirty-file count; prepends a one-line warning when the index lags so the model knows when to distrust results.
  • Agent steering: gcx init now writes a short CLAUDE.md snippet into the target repo's .claude/ directory, nudging the coding agent to prefer graph tools over grep.
  • 3-round median benchmark: stable-v063 replaces the single-run harness — per-(repo,question) median across 3 rounds kills run-to-run noise; geomean rises to 1.74× on 5 canonical repos.
◀ Previous published run — 2026-07-12

The 2026-07-12 run reported up to 56% savings (full MCP, large repos) and 1.15–1.36× compact geomean. Those were single-run numbers; this page supersedes them with median-of-N data and a rate-limit-resilient harness. Full archived report: final-report-2026-07-12.html.

Methodology

Model claude-haiku-4-5. Compact MCP (single dispatch tool). Per question, two Claude sessions (grep arm vs gcx arm), 3 rounds, sequential + throttled with retries. Tokens = input + cache-creation (cache reads excluded to avoid double counting). Errored sessions dropped. Reproduce: bash docs/benchmarks/stable-sweep.sh.
