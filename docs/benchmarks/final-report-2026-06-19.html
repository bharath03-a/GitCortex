Does a code knowledge graph help AI work more efficiently?

Measured Claude token usage on real developer questions, once with ordinary file search, once with the GitCortex graph (compact MCP). This is the median of 3 runs with errored/rate-limited sessions excluded — the noise-resistant view. Every number is from the actual API response.

What shipped in v0.6.0

  • Structural Markdown ingestion. .md/.markdown files indexed as first-class nodes: headings become Section nodes (nested via Contains), inline code-spans matching identifier shape produce References edges to code symbols. Cross-language by design — a README referencing a Rust function resolves correctly.
  • find_god_nodes MCP tool + CLI. Surfaces high-fan-in hub symbols ranked by inbound Calls in-degree. Deterministic across re-runs. Use it to find your God objects before a refactor.
  • find_clusters MCP tool + CLI. Label-propagation community detection over Contains+Calls adjacency. Deterministic, zero new dependencies, no LLM calls. Members per cluster capped at 25; size is always honest.
  • Hub-node overlay in gcx viz. Press G to highlight high-fan-in symbols in cyan. Same centrality analysis as find_god_nodes, exposed visually.
  • Fixed: HTML export XSS. A node name containing </script> (from an untrusted cloned repo) could inject JavaScript into exported HTML files. Fixed with a one-line escape.
  • Fixed: get_subgraph turn explosion. Was consuming 18–27 turns on large repos due to over-verbose prose summaries. Rewritten to stay within the token budget in 4–8 turns.
  • Viz accessibility. F/Space shortcuts wired; ARIA combobox, tablist/tabpanel, radiogroup patterns; contrast fix (--color-text-dim from ~3:1 to ~5.1:1 WCAG AA).
  • Schema version 12 (up from 11). Triggers one-time full re-index on first run after upgrade.

Methodology

Model claude-haiku-4-5. Compact MCP (single dispatch tool). Per question, two Claude sessions (grep arm vs gcx arm), 3 rounds, sequential + throttled with retries. Tokens = input + cache-creation (cache reads excluded to avoid double counting). Errored sessions dropped. Reproduce: bash docs/benchmarks/stable-sweep.sh.
