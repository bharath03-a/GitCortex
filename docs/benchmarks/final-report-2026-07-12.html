What shipped in v0.6.2 / v0.6.3

  • Section filter for Markdown. Section nodes carry their heading level; MCP tools accept an optional section param to scope results to a doc subtree. Enables precise "find all code referenced in the Authentication section" queries.
  • Stop-searching messages. When the token budget is reached, the MCP server emits a structured stop_reason instead of silently truncating — the model knows to summarise rather than keep expanding.
  • Harness fixes. Rate-limit retries, session isolation, and per-question error attribution in the benchmark harness — cleaner multi-round sweeps with fewer phantom errors.
  • refactor(mcp): tools.rs split (v0.6.3). Parameter structs, shared helpers, and git-diff utilities extracted into params.rs, helpers.rs, git_helpers.rs. Canonical FromStr impls for NodeKind and Visibility added to gitcortex-core, eliminating three duplicate string-to-enum match tables. Pure refactor; no behaviour change.

Methodology

Model claude-haiku-4-5. Compact MCP (single dispatch tool). Per question, two Claude sessions (grep arm vs gcx arm). v0.6.2 data is single-round; full median-of-3 sweep in progress. Tokens = input + cache-creation (cache reads excluded to avoid double-counting). Errored sessions dropped per question. Reproduce: bash docs/benchmarks/real-harness.sh.
